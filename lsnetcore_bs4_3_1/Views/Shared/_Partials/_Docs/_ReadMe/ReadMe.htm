MICROSOFT ASP.NET CORE

 

Microsoft ASP.NET Core is a cross-platform application and web framework which can be deployed on the Windows, Linux, and IOS operating systems running ASP.NET Core runtime or hosting.

DEFAULT HOME PAGE SETTINGS:

 

By default, you should see the Bootstrap top navigation bar and the bottom copyright section, included in this readme. The Index.cshtml file is currently configured to display the content of this readme file on the Home page. This serves two purposes:

 

1.  The website is routing to the correct default view and displaying the Home page as intended.

2.  The home page is displaying the necessary document which tests the HomeControler.cs program.

 

Not seeing the page displayed correctly means there is a technical issue with the package.

   

It is highly recomended these instructions be removed from the default Home Page before publishing your

your website. The content of the _ReadMe.cshtml page can be removed from the Home page by modifying the Index.cshtml as described below. Once Index.cshtml file is modified to remove the readme contents the Home page will appear blank and can be used to populate with your own content.

 

USING MICROSOFT VISUAL STUDIO

 

Open the site solution and expand the folders ‘Views\Home’, then open the Index.cshtml View Page.

 

Edit the page to remove the single ‘partial’ line code which loads the readme contents.

 

Save the file then execute the project from Visual Studio, or IIS Express. A blank page means the site is executing properly.

 

 

USING A TEXT EDITOR

 

With the text editor navigate to the site solution directory which should be in the following format…

 

[Your solution directory name]\[project directory name]\Views\Home

 

Open the Index.cshtml file and remove the partial line code which loads the readme contents.

 

Save the file then execute the project from Visual Studio, or IIS Express. A blank page means the site is executing properly.
